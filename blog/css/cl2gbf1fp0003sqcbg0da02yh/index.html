伪类和伪元素详解

2022-04-12 css

伪类

伪类是选择器的一种，它用于选择处于特定状态的元素，比如当它们是这一类型的第一个元素时，或者是当鼠标指针悬浮在元素上面的时候。它们表现得会像是你向你的文档的某个部分应用了一个类一样，帮你在你的标记文本中减少多余的类，让你的代码更灵活、更易于维护

类型 定义 版本
:first-child 匹配在一组兄弟元素中的第一个元素 Selectors Level 4 :first-child
:last-child 匹配在一组兄弟元素中的最后一个元素 css1
:only-child 匹配没有任何兄弟元素的元素 css2
:nth-child(an+b) addad css1
:first-of-type 表示一组兄弟元素中其类型的第一个元素 Selectors Level 4 :first-of-type
:active 匹配被用户激活的元素 HTML Living Standard:active
:auto-fill input css1
:checked 匹配任意选中状态下radio、checkbox、select下的option元素 Selectors Level 4
:default 表示一组相关元素中的默认表单元素 Selectors Level 4 :default
:defined 表示任何已定义的元素,包括任何浏览器内置的标准元素以及已成功定义的自定义元素 HTML Living Standard :defined
:disabled 表示任何被禁用的元素 HTML Living Standard :disabled
:empty 代表没有子元素的元素。子元素只可以是元素节点或文本（包括空格）。注释或处理指令都不会产生影响 Selectors Level 4 :empty
:enabled 表示任何被启用的（enabled）元素 HTML Living Standard :enabled
:first @page CSS 伪类选择器 表示打印文档的时候，第一页的样式 CSS Paged Media Module Level 3 :first

:active

描述：:active 伪类匹配被用户激活的元素，一般就是指鼠标按下时对应的元素，通常用在 <a></a> 和 <button></button> 元素中。

  • button 元素使用 :active 效果 demo
  • a（超链接） 元素使用 :active 效果 demo
    注意 当超链接使用 :active 伪类时, 会被其他的伪类覆盖，这些伪类包括 :link, :visited, :hover。为了保证样式生效，需要吧 :active 放到上面伪类之后。简便记忆方式就是 LVHA, 对应顺序 :link, :visited, :hover, :active。

:autofill

描述：**:autofill** 应用于 input 表单元素，当 input 元素处于 focus 状态时（鼠标点击input元素），并且浏览器展示自动填充选项时，操作这些选项就会命中 :autofill 伪类。

:autofill 可以配合 autocomplete 属性，当 autocomplete 不为 off 时，浏览器会展示自动填充选项。关于 autocomplete 的规则可以参考这里
